# Kế hoạch Tích hợp Supabase (Backend & Database)

Dự án "Triết Gia AI" hiện tại đang chạy hoàn toàn bằng **Mock Data (Dữ liệu giả lập)** trên Frontend (file JSON, biến `const`). Để dự án thực sự là một Web App hoàn chỉnh có thể mở rộng, việc đưa toàn bộ dữ liệu này lên **Supabase (PostgreSQL Database)** là bước đi cực kỳ xuất sắc.

Dưới đây là toàn bộ bản thiết kế Kiến trúc CSDL (Database Schema) và các bước cần thiết để tôi và bạn cùng thực hiện.

## User Review Required

> [!WARNING]
> **Tạo Dự án Supabase**: Tôi không thể tự tạo tài khoản Supabase cho bạn. Bạn cần truy cập [Supabase.com](https://supabase.com), tạo một Project mới (miễn phí), sau đó làm theo hướng dẫn ở phần bên dưới để cung cấp API Keys cho tôi. Bạn đã sẵn sàng làm việc này chưa?

## 1. Kiến trúc Cơ sở dữ liệu (Database Schema)
Dựa trên toàn bộ mã nguồn tôi đã đọc, chúng ta cần tạo **5 Bảng (Tables)** chính trong Supabase.

*Dưới đây là mã SQL. Bạn chỉ cần copy mã này, dán vào mục **SQL Editor** trên Supabase và chạy là xong!*

```sql
-- 1. Bảng Hồ sơ Người dùng (Mở rộng từ Supabase Auth)
create table profiles (
  id uuid references auth.users not null primary key,
  display_name text,
  bio text,
  avatar_url text,
  streak_count integer default 1,
  last_login date default CURRENT_DATE,
  personality_type text
);

-- 2. Bảng Thư viện: Danh mục Triết học
create table categories (
  id text primary key, -- 'ancient', 'modern', v.v.
  title text not null,
  description text,
  icon_name text,
  color text
);

-- 3. Bảng Thư viện: Sách
create table books (
  id bigint generated by default as identity primary key,
  category_id text references categories(id),
  title text not null,
  author text not null,
  type text,
  read_time text,
  summary text,
  sections jsonb -- Dùng kiểu JSONB để chứa toàn bộ nội dung sách (headings, paragraphs, quotes)
);

-- 4. Bảng Câu hỏi Quiz
create table questions (
  id bigint generated by default as identity primary key,
  topic text not null,
  question text not null,
  options jsonb not null, -- Mảng 4 đáp án (A,B,C,D và đáp án đúng)
  explanation text
);

-- 5. Bảng Bài viết/Video (Khám phá)
create table explore_content (
  id bigint generated by default as identity primary key,
  title text not null,
  description text,
  read_time text,
  type text, -- 'video' hoặc 'article'
  featured boolean default false
);

-- 6. Bảng Lịch sử Chat (Trò chuyện AI)
create table chat_messages (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users(id) on delete cascade,
  sender text not null, -- 'user' hoặc 'ai'
  message text not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);
```

## 2. Các bước bạn cần chuẩn bị (Cần thiết)

Để tôi có thể bắt đầu code kết nối Supabase vào App, bạn hãy làm 2 bước sau:

1. **Đăng ký/Đăng nhập Supabase**, tạo một New Project.
2. Vào phần **Project Settings > API**. 
3. Tìm và copy 2 chuỗi bí mật sau:
   - `Project URL`
   - `Project API Keys (anon / public)`
4. Dán chúng vào file `.env` trong dự án của chúng ta:
   ```env
   VITE_SUPABASE_URL=https://xyz.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpX...
   ```

## 3. Lộ trình thực hiện của tôi (Khi bạn ấn "Đồng ý")

Ngay sau khi bạn tạo xong Supabase và ấn Approve, tôi sẽ thực hiện các việc sau:

### Giai đoạn 1: Kết nối & Xác thực (Auth)
- Chạy `npm install @supabase/supabase-js`.
- Tạo file `src/lib/supabase.ts` để khởi tạo kết nối Database.
- Thay đổi trang Đăng nhập (Login) từ "bấm bừa là vào" thành đăng nhập thật bằng Email/Mật khẩu hoặc Google qua Supabase Auth.

### Giai đoạn 2: Migrate Dữ liệu
- Viết một script nhỏ (chạy 1 lần) để đẩy toàn bộ nội dung trong `public/data/library.json` và `QuizPage.tsx` lên Database của bạn.

### Giai đoạn 3: Cập nhật Giao diện
- **LibraryPage**: Sửa hàm `fetch` để gọi dữ liệu thật từ bảng `categories` và `books`.
- **QuizPage**: Thay vì lấy mảng cố định, sẽ gọi `supabase.from('questions').select('*')`.
- **ProfilePage**: Đồng bộ hình ảnh Avatar, Bio và Chuỗi ngày lên Database đám mây thay vì lưu tạm trên máy (localStorage) như hiện tại.
